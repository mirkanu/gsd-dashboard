  <action>
Add a new route in server/routes/gsd.js after the existing POST /projects/:name/send route:

```
POST /api/gsd/projects/create
Body: { name: string, basePath?: string }
```

Implementation steps:
1. Validate `name`: required, non-empty string, must match `/^[a-zA-Z0-9_-]+$/` (no slashes, dots, spaces). Return 400 on failure.
2. Resolve `basePath`: use `req.body.basePath` if provided and is an absolute path, otherwise default to `/data/home`. This makes the base path configurable without an env var.
3. Compute `dir = path.join(basePath, name)`.
4. Load config. Check if a project with that name already exists — return 409 if so.
5. Create directory: `fs.mkdirSync(dir, { recursive: true })`. On error return 500.
6. Create detached tmux session: `execFileSync('tmux', ['new-session', '-d', '-s', name, '-c', dir], { stdio: 'ignore' })`. On error return 500 (directory was created; that's acceptable partial state — user can retry or clean up).
7. Send Claude launch sequence into the session:
   - `execFileSync('tmux', ['send-keys', '-t', name, 'claude', 'Enter'], { stdio: 'ignore' })`
   - Brief pause: `await new Promise(r => setTimeout(r, 500))` — allows claude to start before the slash command is sent
   - `execFileSync('tmux', ['send-keys', '-t', name, '/gsd:new-project', 'Enter'], { stdio: 'ignore' })`
8. Build new config entry: `{ name, root: dir, tmux_session: name }`.
9. Append to config and write back atomically:
   - Read current config with loadConfig()
   - Push new entry to projects array
   - `fs.writeFileSync(configPath, JSON.stringify(config, null, 2), 'utf8')`
   - Use the same `configPath` resolution as loadConfig() — read `process.env.GSD_PROJECTS_PATH || path.resolve(__dirname, '../../gsd-projects.json')`
10. Return 201: `{ ok: true, project: { name, root: dir, tmux_session: name } }`.

GSD_DATA_URL proxy pattern: If `GSD_DATA_URL` is set, forward to upstream as POST with same body. Return upstream status + body.

The route is: `router.post('/projects/create', async (req, res) => { ... })`.

Do NOT use execSync (use execFileSync to avoid shell injection). Do NOT accept names with special characters.
