  <name>Task 2: Add api.gsd.file() method to api client</name>
  <files>client/src/lib/api.ts</files>
  <action>
    The existing `request<T>()` helper always parses JSON and sets Content-Type: application/json. The file endpoint returns `text/plain`, so a separate text-mode helper is needed.

    Add a `requestText()` helper immediately after the existing `request<T>()` function (before the `export const api` block):

    ```typescript
    async function requestText(path: string): Promise<string> {
      const res = await fetch(`${BASE}${path}`);
      if (!res.ok) {
        const body = await res.json().catch(() => ({}));
        throw new Error(body?.error || `HTTP ${res.status}`);
      }
      return res.text();
    }
    ```

    Then extend the `gsd` namespace inside `export const api` to add a `file` method:

    ```typescript
    gsd: {
      projects: () => request<{ projects: import("./types").GsdProject[] }>("/gsd/projects"),
      file: (projectName: string, fileId: "state" | "roadmap" | "requirements" | "plan") =>
        requestText(`/gsd/projects/${encodeURIComponent(projectName)}/files/${fileId}`),
    },
    ```

    Do NOT change the `request<T>()` helper, the `BASE` constant, or any other api namespace. Do NOT change the `gsd.projects` method.
  </action>
