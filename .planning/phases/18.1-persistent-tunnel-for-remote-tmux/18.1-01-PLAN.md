<objective>
Replace the ephemeral Cloudflare Quick Tunnel (trycloudflare.com) with a named Cloudflare Tunnel that has a permanent, stable subdomain. The URL will never rotate, so Railway's GSD_DATA_URL only needs to be set once — not updated on every restart.

Purpose: The existing tunnel.sh calls Railway's GraphQL API and triggers a redeploy on every tunnel restart, causing 502 errors and dashboard instability. A named tunnel with a fixed URL eliminates this problem entirely.

Output: Updated tunnel.sh, a systemd service unit file, and a one-time setup script.
</objective>

Rewrite scripts/tunnel.sh to use a named Cloudflare Tunnel instead of an ephemeral quick tunnel. The file must be POSIX sh (#!/usr/bin/env sh), not bash — consistent with the Alpine Docker base.

New behavior:
1. Read CLOUDFLARE_TUNNEL_NAME from env (required — exit with error if missing)
2. Read DASHBOARD_PORT from env, default 4820
3. Set LOG_FILE to /tmp/gsd-tunnel.log
4. Define a log() function: echo "[timestamp] $*" | tee -a $LOG_FILE
5. Run a while-true loop:
   a. Log "Starting named tunnel '$CLOUDFLARE_TUNNEL_NAME' -> localhost:$DASHBOARD_PORT"
   b. Run: cloudflared tunnel --no-autoupdate run --url "http://localhost:$DASHBOARD_PORT" "$CLOUDFLARE_TUNNEL_NAME"
      - This blocks until the tunnel dies
      - cloudflared exits non-zero on failure
   c. Log "Tunnel exited (code $?). Restarting in 15s..."
   d. sleep 15

Remove ALL of the following (no longer needed):
- RAILWAY_PROJECT_ID, RAILWAY_SERVICE_ID, RAILWAY_ENV_ID variables
- RAILWAY_GQL and gql() function
- update_railway() function
- The 30-second URL-capture polling loop (grep for trycloudflare.com URL)
- Background cloudflared with TMP_LOG capture
- The Railway API token loading from .env

The .env loading and DASHBOARD_PORT reading can remain (CLOUDFLARE_TUNNEL_NAME may come from .env).

<task type="auto">
  <name>Task 2: Add systemd service file and one-time setup script</name>
  <files>scripts/tunnel.service, scripts/tunnel-setup.sh</files>
  <action>
Create two new files:

**scripts/tunnel.service** — systemd unit for auto-starting the tunnel on boot:
```
[Unit]
Description=GSD Dashboard Cloudflare Tunnel
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
EnvironmentFile=-/data/home/gsddashboard/.env
ExecStart=/data/home/gsddashboard/scripts/tunnel.sh
Restart=always
RestartSec=15
StandardOutput=append:/tmp/gsd-tunnel.log
StandardError=append:/tmp/gsd-tunnel.log

[Install]
WantedBy=multi-user.target
```
Notes on the service file:
- EnvironmentFile path must be absolute and match the actual .env location
- Use `EnvironmentFile=-` (with leading dash) so systemd continues if .env is missing
- The script path in ExecStart must be absolute
- Restart=always with RestartSec=15 supplements the retry loop in tunnel.sh

**scripts/tunnel-setup.sh** — one-time guided setup script (POSIX sh, chmod +x):

The script should print clear step-by-step instructions (using echo) for:
1. Step 1: Login — `cloudflared login` (opens browser for OAuth, stores cert.pem)
2. Step 2: Create tunnel — `cloudflared tunnel create gsd-local` (creates UUID credentials)
3. Step 3: Route DNS — `cloudflared tunnel route dns gsd-local YOUR_HOSTNAME` (e.g. gsd-local.yourdomain.com)
4. Step 4: Add to .env — `echo "CLOUDFLARE_TUNNEL_NAME=gsd-local" >> .env`
5. Step 5: Set Railway env var — one-time only: `GSD_DATA_URL=https://YOUR_HOSTNAME`
6. Step 6: Install systemd service (optional) — copy unit file and enable

The script should NOT try to run cloudflared commands automatically (cloudflared login requires an interactive browser). It should only print the commands with explanations so the user can run them manually.

Add a final section printing: "Once the tunnel is running, set GSD_DATA_URL on Railway to the permanent hostname. This only needs to be done ONCE — the URL never changes."
