**Named Cloudflare Tunnel replacing ephemeral quick tunnel — permanent URL eliminates Railway API redeploy loop on every tunnel restart**

## Accomplishments
- Rewrote tunnel.sh: removed all Railway GraphQL API calls and ephemeral URL capture; replaced with named `cloudflared tunnel run` inside a self-healing while loop
- Created tunnel.service: systemd unit with EnvironmentFile, Restart=always, RestartSec=15, and log appending
- Created tunnel-setup.sh: step-by-step printed guide for one-time login, tunnel creation, DNS routing, .env setup, Railway env var, and optional systemd install

## Files Created/Modified
- `scripts/tunnel.sh` - Named tunnel launcher: POSIX sh, reads CLOUDFLARE_TUNNEL_NAME, self-healing while loop, no Railway API
- `scripts/tunnel.service` - Systemd unit pointing to absolute paths, EnvironmentFile with leading dash (optional load)
- `scripts/tunnel-setup.sh` - Printed one-time setup guide (login, create tunnel, route DNS, .env, Railway, systemd install)

## Decisions Made
- Named tunnel replaces ephemeral quick tunnel: GSD_DATA_URL on Railway is set once and never changes, eliminating the redeploy loop that caused 502 errors
- tunnel-setup.sh intentionally does not run cloudflared automatically — `cloudflared login` requires an interactive browser OAuth flow
- POSIX sh throughout for Alpine Docker compatibility (the existing tunnel.sh was incorrectly using bash)
- Restart delay reduced to 15s (was 30s) — faster recovery while still giving cloudflared time to settle

None - plan executed exactly as written.

None.

## User Setup Required

Before running the new tunnel.sh, one-time setup is required:

1. Run `sh scripts/tunnel-setup.sh` and follow each printed step
2. Add `CLOUDFLARE_TUNNEL_NAME=<your-tunnel-name>` to `.env`
3. Set `GSD_DATA_URL=https://<your-hostname>` in the Railway dashboard (one-time only)

After that, start the tunnel with `sh scripts/tunnel.sh` or install the systemd service.
