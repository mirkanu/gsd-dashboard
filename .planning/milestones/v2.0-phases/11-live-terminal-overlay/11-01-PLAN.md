  <action>
Install node-pty as an optional dependency (same pattern as better-sqlite3 — optional so Railway builds don't fail if native compilation is unavailable):

```
npm install --save-optional node-pty
```

Create server/routes/terminal.js that exports a single function `attachTerminalWS(server)`. This function:

1. Creates a `new WebSocketServer({ noServer: true })` (no path, no port — attached via upgrade event).

2. The caller (index.js) wires the upgrade event. attachTerminalWS attaches to the http.Server's `upgrade` event. Inside the upgrade handler:
   - If `req.url` does NOT start with `/ws/terminal/` → do nothing (let other handlers process it, e.g. the main /ws endpoint).
   - Parse the project name from the URL: `req.url.replace(/^\/ws\/terminal\//, '').split('?')[0]` then `decodeURIComponent(...)`.
   - Load gsd-projects.json via `loadConfig()` (reuse the same helper used in routes/gsd.js — extract it or require it from the same module; if it is not exported yet, inline the same read logic: `JSON.parse(fs.readFileSync(process.env.GSD_PROJECTS_PATH || path.join(__dirname, '../../gsd-projects.json'), 'utf8'))`).
   - Find the project entry by name; if not found, destroy the socket and return.
   - Check `isTmuxSessionActive(project.tmux_session)`. If false, emit a WS close with code 4004 and reason "session inactive": use `wss.handleUpgrade(req, socket, head, (ws) => { ws.close(4004, 'session inactive'); })` then return.
   - Otherwise call `wss.handleUpgrade(req, socket, head, (ws) => { ... pty bridge logic ... })`.

3. PTY bridge logic inside the handleUpgrade callback:
   - Spawn the pty: `const pty = require('node-pty').spawn('tmux', ['attach-session', '-t', session, '-d'], { name: 'xterm-256color', cols: 80, rows: 24, cwd: process.env.HOME || '/' })`.
     - Flag `-d` is NOT used here — use `tmux attach-session -t <session>` WITHOUT `-d` so we attach without detaching other clients (dashboard should be a non-destructive observer). Remove the `-d` flag.
   - Forward pty output to WebSocket: `pty.onData((data) => { if (ws.readyState === 1) ws.send(data); })`.
   - Forward WebSocket messages to pty: `ws.on('message', (msg) => { ... })`. Messages are either:
     - A JSON string `{ type: 'resize', cols: N, rows: N }` → call `pty.resize(cols, rows)`.
     - Any other string → write to pty: `pty.write(data.toString())`.
     - Handle JSON parse errors: if JSON.parse fails, treat the entire message as raw input (write to pty).
   - On pty exit: `pty.onExit(() => { if (ws.readyState === 1) ws.close(1000, 'pty exited'); })`.
   - On ws close: `ws.on('close', () => { try { pty.kill(); } catch {} })`.

4. node-pty may not be available in all environments (optional dep). Wrap the `require('node-pty')` inside the handleUpgrade callback. If it throws (module not found), close the WS with code 4005 and reason 'node-pty unavailable' instead of crashing the server.

Export: `module.exports = { attachTerminalWS };`
