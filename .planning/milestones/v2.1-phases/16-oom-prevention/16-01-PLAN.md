# Plan 16-01: Node.js Memory Cap + Memory Watchdog

## Context

The `claude-code-ssh` Railway container (the SSH host where Claude Code sessions run) crashed with OOM. The user runs 4 concurrent Claude Code sessions in tmux, each spawning multiple node processes with the default 2.2GB heap limit. Over time, orphaned processes accumulate and memory usage grows unbounded. Currently there are zero memory limits, zero cleanup, and zero monitoring.

## Goal

Prevent OOM crashes by: (1) capping Node.js heap per process, (2) running a background watchdog that cleans up orphans and force-kills when memory is critical, (3) auto-starting the watchdog on login.

---

## Tasks

### Task 1: Set NODE_OPTIONS globally

**File:** `/etc/profile.d/claude-code.sh`

Add `NODE_OPTIONS` export to cap every node process at 1GB heap:

```bash
export NODE_OPTIONS="--max-old-space-size=1024"
```

Append to existing file (which currently only sets PATH and locale).

**Why 1024MB:** With 4 sessions × ~3 node processes each = ~12 processes. At 1GB cap = 12GB max theoretical, well within Railway container limits. The default 2.2GB × 12 = 26GB+ which causes OOM.

### Task 2: Create memory watchdog script

**File:** `/data/home/.local/bin/memory-watchdog.sh`

Create executable script that:

1. Loops every 5 minutes (`sleep 300`)
2. Reads `/proc/meminfo` to calculate memory usage percentage
3. Logs current usage to `/tmp/memory-watchdog.log`
4. **At 80% usage:** Warns in log, finds orphaned node processes (node processes whose parent PID is 1, i.e. reparented to init — these are orphans), kills them with SIGTERM
5. **At 90% usage:** Finds the largest node process by RSS (excluding PID of the current Claude Code session in the active tmux pane), kills it with SIGKILL, logs what was killed
6. Traps SIGTERM for clean shutdown

```bash
#!/usr/bin/env bash
# Memory watchdog — prevents OOM on claude-code-ssh
# Runs every 5 minutes, cleans orphaned node processes at 80%,
# force-kills largest node at 90%.

set -euo pipefail

LOG="/tmp/memory-watchdog.log"
INTERVAL=300
WARN_PCT=80
CRIT_PCT=90

log() { echo "[$(date -u +%FT%TZ)] $*" >> "$LOG"; }

get_mem_pct() {
  awk '/^MemTotal:/{t=$2} /^MemAvailable:/{a=$2} END{printf "%d", (t-a)*100/t}' /proc/meminfo
}

kill_orphan_nodes() {
  # Find node processes whose parent is PID 1 (orphaned/reparented)
  local killed=0
  for pid in $(pgrep -x node 2>/dev/null); do
    ppid=$(ps -o ppid= -p "$pid" 2>/dev/null | tr -d ' ')
    if [[ "$ppid" == "1" ]]; then
      cmd=$(ps -o args= -p "$pid" 2>/dev/null || echo "unknown")
      log "WARN: killing orphaned node (PID $pid): $cmd"
      kill "$pid" 2>/dev/null || true
      ((killed++))
    fi
  done
  [[ $killed -gt 0 ]] && log "Killed $killed orphaned node process(es)"
}

kill_largest_node() {
  # Find largest node process by RSS, kill it
  local target_pid target_rss target_cmd
  target_pid=$(ps -eo pid,rss,comm --sort=-rss | awk '$3=="node"{print $1; exit}')
  if [[ -n "$target_pid" ]]; then
    target_rss=$(ps -o rss= -p "$target_pid" 2>/dev/null | tr -d ' ')
    target_cmd=$(ps -o args= -p "$target_pid" 2>/dev/null || echo "unknown")
    log "CRIT: force-killing largest node (PID $target_pid, ${target_rss}kB): $target_cmd"
    kill -9 "$target_pid" 2>/dev/null || true
  fi
}

log "Watchdog started (warn=${WARN_PCT}%, crit=${CRIT_PCT}%, interval=${INTERVAL}s)"

while true; do
  pct=$(get_mem_pct)
  log "Memory: ${pct}% used"

  if (( pct >= CRIT_PCT )); then
    log "CRITICAL: ${pct}% memory — killing largest node process"
    kill_orphan_nodes
    kill_largest_node
  elif (( pct >= WARN_PCT )); then
    log "WARNING: ${pct}% memory — cleaning orphaned nodes"
    kill_orphan_nodes
  fi

  sleep "$INTERVAL"
done
```

### Task 3: Auto-start watchdog on SSH login

**File:** `/data/home/.bashrc`

Append a guard block that starts the watchdog if not already running:

```bash
# Memory watchdog — prevent OOM on claude-code-ssh
if ! pgrep -f "memory-watchdog.sh" >/dev/null 2>&1; then
  nohup "$HOME/.local/bin/memory-watchdog.sh" </dev/null &>/dev/null &
  disown
fi
```

Uses `pgrep -f` to avoid duplicates. `nohup` + `disown` ensures it survives session disconnect.
