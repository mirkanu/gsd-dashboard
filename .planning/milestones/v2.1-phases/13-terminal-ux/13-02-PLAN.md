<objective>
Fix mobile keyboard occlusion and enable touch scroll in the terminal overlay.

Purpose: On mobile, the software keyboard overlays a fixed-position element without shifting it. This hides the send box. The fix uses the `visualViewport` API to detect the available height and shift the overlay up. Touch scroll also requires an explicit `touch-action` hint to pass swipe events through to xterm.js.

Output:
- `visualViewport` resize listener in `TerminalOverlay` that sets a `bottomOffset` state and applies it as `bottom` inline style on the overlay root
- `fitAddon.fit()` call triggered after each viewport resize
- `touch-action: pan-y` on the terminal container div
- `overscrollBehavior: 'contain'` on the overlay root to prevent scroll bleed
</objective>

  <action>
Two changes to `TerminalOverlay`:

**A. Add `bottomOffset` state** at the top of the function body (alongside existing refs):
```tsx
const [bottomOffset, setBottomOffset] = useState(0);
```

**B. Add a second `useEffect`** (separate from the terminal setup effect, so it doesn't re-run when `projectName` changes and accidentally reconnect the terminal). Place it after the existing `useEffect`:
```tsx
useEffect(() => {
  if (!window.visualViewport) return;

  const handleViewportResize = () => {
    const offset = window.innerHeight - (window.visualViewport?.height ?? window.innerHeight);
    setBottomOffset(Math.max(0, offset));
    // Re-fit terminal columns to new available height
    if (fitAddonRef.current) {
      // Small delay so the DOM has reflowed before measuring
      setTimeout(() => fitAddonRef.current?.fit(), 50);
    }
  };

  window.visualViewport.addEventListener('resize', handleViewportResize);
  return () => {
    window.visualViewport?.removeEventListener('resize', handleViewportResize);
  };
}, []); // empty deps — mount/unmount only
```

**C. Apply `bottomOffset` to the overlay root's inline style**. Change the root div's `style` from:
```tsx
style={{ zIndex: 70 }}
```
to:
```tsx
style={{ zIndex: 70, bottom: bottomOffset > 0 ? bottomOffset : undefined }}
```

When `bottomOffset` is 0 (no keyboard), `bottom` is `undefined` so `inset-0` (which sets `bottom: 0`) from Tailwind takes full effect unchanged. When the keyboard is open, the explicit `bottom` pushes the overlay up by the exact keyboard height.
