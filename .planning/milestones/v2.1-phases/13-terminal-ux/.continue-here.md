### Post-execution mobile fixes (all shipped, all deployed)
These were not in the original plans but required to make mobile usable:

1. **New tab on mobile** — `window.matchMedia('(pointer: coarse)')` detects touch; "Open terminal" opens `/terminal/:name` in a new browser tab (clean full-screen context) instead of the overlay
2. **TerminalPage component** — exported from GSD.tsx, routed at `/terminal/:name` outside the Layout wrapper in App.tsx
3. **Session not active fix** — TerminalPage showed "Session not active" because it mounted TerminalOverlay before `wsBase` was fetched; now waits with "Connecting…" until wsBase resolves
4. **xterm.js upgrade** — `@xterm/xterm` → `6.1.0-beta.195`, `@xterm/addon-fit` → `0.12.0-beta.195`; beta has touch scroll fix (PR #5563)
5. **onData filter** — filters `\x1b[M` and `\x1b[<` sequences before forwarding to pty; xterm.js beta generates NaN-coordinate mouse sequences from touch events that cause garbage characters
6. **Touch scroll** — passive `touchstart`/`touchmove` listeners send valid SGR sequences (`\x1b[<64;1;1M` scroll up, `\x1b[<65;1;1M` scroll down) directly via `ws.send()` bypassing the onData filter; tmux handles actual scroll
7. **Tap to type** — `touchend` listener calls `terminal.focus()` when no scroll intent; opens iOS soft keyboard
8. **Scroll-to-bottom on keyboard open** — after `fitAddon.fit()`, calls `terminal.scrollToBottom()` so cursor stays visible; sends pty resize message
9. **iOS caret hidden** — `.xterm-helper-textarea { caret-color: transparent }` in index.css
10. **iOS zoom fix** — `.xterm-helper-textarea { font-size: 16px }` prevents iOS auto-zoom on focus (iOS zooms inputs < 16px)
11. **Mobile font size** — 10px on touch devices, 14px on desktop

### Key architectural decisions
- Touch scroll works via pty mouse sequences (tmux handles scrollback), NOT via `terminal.scrollLines()` — the xterm.js local scrollback buffer is mostly empty when attached to tmux
- `capture: true` on touch handlers was tried and rejected — it prevented xterm.js from handling taps (keyboard focus)
- `touch-action: none` was tried and rejected for the same reason
- All touch listeners are `passive: true`; scroll sequences go directly to `ws.send()` not through `onData`
