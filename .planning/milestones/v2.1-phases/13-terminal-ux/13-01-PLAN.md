<objective>
Move SendBox into TerminalOverlay and remove it from ProjectCard.

Purpose: The send input and GSD command chips belong in the terminal overlay for focused, context-aware interaction. Having them on the card creates a split UX where the user types commands separately from the terminal view.

Output:
- `TerminalOverlay` grows an `initialSendValue` prop and renders `<SendBox>` below the xterm.js container
- `ProjectCard`'s `onOpenTerminal` callback signature changes to `(initialValue: string) => void` so `state?.next_action` is threaded through
- `SendBox` render is removed from `ProjectCard`
- The GSD page wires `initialSendValue` into `TerminalOverlay` via state
</objective>

Three changes inside `TerminalOverlay`:

**A. Extend the props interface** to include `initialSendValue`:
```tsx
interface TerminalOverlayProps {
  projectName: string;
  wsBase: string | null;
  onClose: () => void;
  initialSendValue: string;
}
```

**B. Destructure the new prop** in the function signature:
```tsx
function TerminalOverlay({ projectName, wsBase, onClose, initialSendValue }: TerminalOverlayProps) {
```

**C. Update the JSX return** to add SendBox below the terminal container div. The overlay root is already `flex flex-col`; the terminal container is `flex-1`. Add `SendBox` after it with `flex-shrink-0` so it never steals height from the terminal:
```tsx
{/* Terminal container — fills remaining height */}
<div ref={containerRef} className="flex-1 overflow-hidden p-2" />
{/* Send box — pinned below terminal */}
<div className="flex-shrink-0">
  <SendBox
    projectName={projectName}
    initialValue={initialSendValue}
    contextTokens={null}
  />
</div>
```

The `contextTokens={null}` keeps the ContextBar hidden (already commented out in SendBox) and avoids needing to pass token data through.

**D. Call `terminal.focus()` after `terminal.open()`** so the terminal captures keystrokes immediately on desktop without requiring a click:
```tsx
terminal.open(containerRef.current);
fitAddon.fit();
terminal.focus(); // give keyboard focus to terminal on open
```

On mobile this is a no-op (the send input will capture focus via normal tap behavior).

Four targeted edits to wire the initial value through and clean up the card:

**A. Change `onOpenTerminal` prop type in ProjectCard** from `() => void` to `(initialValue: string) => void`:
```tsx
onOpenTerminal: (initialValue: string) => void;
```

**B. Update the "Open terminal" button's onClick** inside ProjectCard to pass `state?.next_action ?? ""`:
```tsx
onClick={(e) => { e.stopPropagation(); onOpenTerminal(state?.next_action ?? ""); }}
```
Note: `state` is already destructured at the top of `ProjectCard` via `const { state, roadmap, requirements } = project;`.

**C. Remove the SendBox render block from ProjectCard.** Delete these lines entirely (both cards — active and archived share the same component, so one deletion covers both):
```tsx
{/* Send box — only when tmux session is active */}
{project.tmuxActive && (
  <SendBox
    projectName={project.name}
    initialValue={state?.next_action ?? ""}
    contextTokens={project.contextTokens ?? null}
  />
)}
```

**D. In the GSD() component**, add state for the terminal initial value and update all `onOpenTerminal` call sites:

Add state declaration alongside the other terminal state:
```tsx
const [terminalInitialValue, setTerminalInitialValue] = useState<string>("");
```

Update both `onOpenTerminal` props (active cards grid and archived cards grid) from:
```tsx
onOpenTerminal={() => setTerminalProject(project.name)}
```
to:
```tsx
onOpenTerminal={(initialValue) => {
  setTerminalProject(project.name);
  setTerminalInitialValue(initialValue);
}}
```

Pass the value into `TerminalOverlay`:
```tsx
<TerminalOverlay
  projectName={terminalProject}
  wsBase={terminalWsBase}
  onClose={() => setTerminalProject(null)}
  initialSendValue={terminalInitialValue}
/>
```

Also reset `terminalInitialValue` when the overlay closes to avoid stale values:
```tsx
onClose={() => { setTerminalProject(null); setTerminalInitialValue(""); }}
```
