<objective>
Add mobile-only special key buttons, hide send UI on desktop, and expand send box on mobile focus.

Purpose: Mobile users cannot type escape sequences, arrow keys, Tab, or Ctrl+C on a software keyboard. The special key bar gives them access to these essential terminal keys. Desktop users already have a physical keyboard, so the send box and chips are unnecessary clutter in the overlay.

Output:
- `SpecialKeyBar` component renders a row of special key buttons that send escape sequences via WebSocket
- `TerminalOverlay` conditionally shows SendBox + SpecialKeyBar on mobile only (detected via `pointer: coarse`)
- On desktop, only the xterm.js terminal is shown in the overlay (no send box, no chips)
- On mobile, focusing the send input expands it to a full-width overlay above the keyboard
</objective>

  <files>client/src/pages/GSD.tsx</files>
  <action>
Add a new `SpecialKeyBar` component after the `SendBox` component definition (around line 202). This component takes a `wsRef` and renders a row of special key buttons:

```tsx
const SPECIAL_KEYS = [
  { label: "←", seq: "\x1b[D" },
  { label: "→", seq: "\x1b[C" },
  { label: "↑", seq: "\x1b[A" },
  { label: "↓", seq: "\x1b[B" },
  { label: "Tab", seq: "\t" },
  { label: "Esc", seq: "\x1b" },
  { label: "Ctrl+C", seq: "\x03" },
  { label: "Enter", seq: "\r" },
] as const;

function SpecialKeyBar({ wsRef }: { wsRef: React.RefObject<WebSocket | null> }) {
  const send = (seq: string) => {
    if (wsRef.current?.readyState === WebSocket.OPEN) {
      wsRef.current.send(seq);
    }
  };
  return (
    <div className="flex flex-wrap gap-1.5 px-4 py-2 border-t border-border/50">
      {SPECIAL_KEYS.map((key) => (
        <button
          key={key.label}
          onTouchStart={(e) => { e.preventDefault(); send(key.seq); }}
          className="text-[11px] px-2.5 py-1.5 rounded border border-border bg-surface-3 text-gray-400 active:bg-accent/20 active:text-accent active:border-accent/30 transition-colors select-none"
        >
          {key.label}
        </button>
      ))}
    </div>
  );
}
```

Key design decisions:
- `onTouchStart` with `preventDefault()` prevents the button from stealing focus from xterm.js and avoids the 300ms touch delay
- `active:` pseudo-class provides visual feedback on press
- `select-none` prevents text selection on long-press
- Escape sequences match standard VT100/xterm sequences that tmux understands
