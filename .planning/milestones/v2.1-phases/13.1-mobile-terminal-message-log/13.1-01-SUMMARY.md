# Phase 13.1 Plan 01: Mobile Terminal Special Keys & Send UI Summary

Mobile special key bar with 8 terminal keys, desktop-only xterm (no send UI), and full-screen send input on mobile focus.

## What Was Done

### Task 1: SpecialKeyBar Component
- Added `SPECIAL_KEYS` constant with 8 key definitions (arrows, Tab, Esc, Ctrl+C, Enter) and their VT100 escape sequences
- Created `SpecialKeyBar` component that renders a row of touch-friendly buttons
- Each button uses `onTouchStart` with `preventDefault()` to avoid stealing focus from xterm.js and eliminate the 300ms touch delay
- Buttons send escape sequences directly via `wsRef.current.send()`

### Task 2: Mobile-Only Send UI in TerminalOverlay
- Added `isMobile` state to `TerminalOverlay` using `useState(() => window.matchMedia('(pointer: coarse)').matches)`
- Wrapped SendBox + SpecialKeyBar in `{isMobile && (...)}` conditional
- On desktop, only the xterm.js terminal is shown (no send box, no chips, no special keys)
- On mobile, send box renders with GSD chips replaced by special key bar below

### Task 3: Full-Screen Send Input on Mobile Focus
- Added `focused` state and `isMobile` detection to `SendBox` component
- When focused on mobile, the send box container gets `fixed left-0 right-0 bottom-0 bg-[#0d1117] z-[80]` classes
- This pins it full-width at the bottom of the screen, above the software keyboard
- On desktop or when unfocused, renders normally in flow
